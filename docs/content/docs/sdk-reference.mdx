Two MIT packages:

- **`@chimely/client`** — headless TypeScript core. Zero runtime dependencies.
  Manages the SSE connection, conditional REST refetch, and optimistic
  mutations.
- **`@chimely/react`** — `<Inbox />`, hooks, and a provider. Runtime UI dep:
  `@floating-ui/dom`.

Wire types in `@chimely/client` are generated from the OpenAPI spec with
`openapi-typescript` (`packages/client/src/generated/`) and never hand-edited.

<Callout type="info" title="Pre-release">
Both packages are at `0.0.0` and not yet on npm. Inside this monorepo they
resolve via `workspace:*`; `npm i @chimely/react` works only after the first
published release.
</Callout>

```bash
npm i @chimely/react react react-dom
```

## `<Inbox />`

A bell + popover. Use it standalone (pass connection props; it owns a client)
or inside a [`ChimelyProvider`](#chimelyprovider) (omit connection props; it
uses the context client).

| Prop | Type | Default | Notes |
|---|---|---|---|
| `serverUrl` | `string` | — | Standalone mode (with `environment`, `subscriberId`) |
| `environment` | `string` | — | Environment slug |
| `subscriberId` | `string` | — | Your subscriber id |
| `subscriberHash` | `string` | — | HMAC hash; required in production environments |
| `backoff` | `BackoffConfig` | — | SSE reconnect backoff |
| `appearance` | `InboxAppearance` | — | CSS-variable theming + per-slot classNames |
| `localization` | `Partial<InboxLocalization>` | — | UI text overrides |
| `tabs` | `InboxTab[]` | — | Tab strip between header and list; see below |
| `placement` | `'bottom-start' \| 'bottom-end' \| 'top-start' \| 'top-end'` | `'bottom-end'` | Popover position |
| `placementOffset` | `number` | `8` | Distance in px between bell and popover |
| `portal` | `boolean` | `false` | Render the popover in a `document.body` portal (fixed positioning); escapes `overflow` and `transform` ancestors |
| `open` | `boolean` | — | Controlled open state; omit for self-managed |
| `onOpenChange` | `(open: boolean) => void` | — | Fires on every open/close intent, controlled or not |
| `routerPush` | `(url: string) => void` | — | SPA navigation for same-origin action URLs (called with pathname + search + hash); cross-origin still full-navigates |
| `preferencesPanel` | `boolean` | `true` | Show per-category toggles |
| `onItemClick` | `(item) => boolean \| void` | — | Return `false` to skip default mark-read + navigate |
| `renderItem` | `(ctx) => ReactNode` | — | Full item override, including click wiring |
| `renderSubject` | `({ item }) => ReactNode` | — | Replace only the subject line; layout and click wiring stay |
| `renderBody` | `({ item }) => ReactNode` | — | Replace only the body line |
| `renderAvatar` | `({ item }) => ReactNode` | — | Replace only the leading icon/avatar |
| `renderBell` | `(ctx) => ReactNode` | — | Custom bell/badge renderer |
| `renderEmpty` | `() => ReactNode` | — | Custom empty state |
| `renderFooter` | `() => ReactNode` | — | Content for the popover footer (branding, links) |

### Appearance

`appearance.variables` forwards keys as `--chimely-<key>` CSS custom
properties (`colorPrimary`, `colorBackground`, `colorBadge`,
`colorBadgeForeground`, `shadow`, `borderRadius`, `fontFamily`, …).
`appearance.classNames` adds classes per slot and `appearance.styles` adds
inline styles per slot: `root`, `bell`, `badge`, `popover`, `content`,
`header`, `tabs`, `tab`, `tabActive`, `list`, `item`, `itemUnread`, `empty`,
`footer`, `pill`, `preferences`. `appearance.icons` replaces the built-in
SVGs (`bell`, `gear`); `renderBell` wins over `icons.bell`.

A dark preset ships with the package. Spread it and override freely:

```tsx
import { darkTheme } from '@chimely/react';

<Inbox appearance={{ variables: { ...darkTheme, colorPrimary: '#8b5cf6' } }} />
```

When notifications arrive while the list is scrolled down, a "N new
notifications" pill overlays the list top instead of yanking the viewport;
clicking it scrolls back up. The text comes from
`localization.newNotifications`.

### Read state and views

Hovering a row reveals a read/unread toggle; items marked unread stay unread
even across "Mark all as read" (the override survives the watermark). The
header's view select switches between the full inbox and unread-only, a
server-side filter that paginates independently.

The toggle buttons draw their glyph in `colorPrimary` on `colorBackground`
with a `colorMuted` border, and each carries a `title` and `aria-label`. Both
built-in presets keep the glyph above the WCAG contrast thresholds. Custom
themes should keep `colorPrimary` legible against `colorBackground` (3:1 or
better).

### Tabs

```tsx
<Inbox
  tabs={[
    { label: 'All' },
    { label: 'Transactions', icon: <MoneyWavy />, filter: (item) => item.payload.tags?.includes('transactions') },
    { label: 'Business', filter: (item) => item.category.startsWith('business.') },
  ]}
/>
```

`InboxTab.filter` is a client-side predicate over loaded items: it can match
tag arrays, category prefixes, or anything else in the payload. A tab without
a filter shows everything. When the active tab's filtered view runs short, the
list keeps fetching pages until the tab fills or the inbox is exhausted.

Per-tab unread counts cover loaded pages only, so they are approximate for
deep inboxes. Server-side filtered counts are tracked in issue #39.

### Localization

Every user-visible string routes through `localization`. All fields optional;
omitted keys keep the `en-US` defaults.

| Key | Default | Notes |
|---|---|---|
| `emptyTitle` | `No notifications` | |
| `emptyBody` | `You're all caught up.` | |
| `markAllRead` | `Mark all as read` | |
| `preferencesTitle` | `Notification preferences` | Preferences header and gear label |
| `inboxTitle` | `Notifications` | List-view header title and dialog label |
| `bellLabel` | `Notifications` | Bell button aria-label |
| `backLabel` | `Back` | Preferences back button aria-label |
| `categoryLabels` | `{}` | Display names for category keys in the preferences panel |
| `filterLabel` | `View` | View select aria-label |
| `filterInbox` | `Inbox` | Default view option |
| `filterUnread` | `Unread` | Unread-only view option |
| `markReadAction` | `Mark as read` | Per-item action tooltip |
| `markUnreadAction` | `Mark as unread` | Per-item action tooltip |
| `formatTimestamp` | relative time | `(iso) => string` for item timestamps |
| `newNotifications` | `N new notifications` | `(count) => string` |

The default timestamp rendering is relative (`5 minutes ago`, via
`Intl.RelativeTimeFormat`), falling back to the locale date past seven days.
The absolute time stays on the `<time>` element's `dateTime` and `title`
attributes.

## Composable components

Between the drop-in `<Inbox />` and the raw hooks sit three composable
components. All read the client from the enclosing `ChimelyProvider` and
accept `appearance` and `localization`.

- **`<InboxContent />`** is the popover interior: header, notification list,
  preferences view, footer. Put it inside your own popover, drawer, or a
  full-page inbox route. It takes the same item props as `<Inbox />`
  (`onItemClick`, `routerPush`, `preferencesPanel`, and all the render props
  except `renderBell`).
- **`<Bell />`** is the bell button with the unseen badge, `forwardRef` so
  your popover library can anchor to it. Props: `open`, `onClick`, `popupId`
  (aria wiring), `renderBell`. It never calls `markAllSeen`: clearing the
  badge on open is the popover's contract, so call `client.markAllSeen()`
  when your container opens.
- **`<Preferences />`** is the per-category toggle list, for settings pages.

```tsx
import { Bell, InboxContent, useChimelyClient } from '@chimely/react';

function CustomInbox() {
  const client = useChimelyClient();
  const [open, setOpen] = useState(false);
  useEffect(() => {
    if (open) void client.markAllSeen();
  }, [open, client]);
  return (
    <MyPopover trigger={<Bell open={open} onClick={() => setOpen(!open)} />} open={open}>
      <InboxContent />
    </MyPopover>
  );
}
```

## Hooks

Hooks read the client from a `ChimelyProvider`. The snapshot is immutable with
a new identity per change (safe for `useSyncExternalStore`).

```tsx
useNotifications(options?: { pageSize?: number }): {
  items: ReadonlyArray<InboxItem>;
  isLoading: boolean;
  error: ChimelyError | null;
  hasMore: boolean;
  lastRefreshNewItemIds: ReadonlyArray<InboxItemId>;
  filter: InboxFilterView;                  // 'default' | 'unread'
  fetchMore: () => Promise<void>;
  refresh: () => Promise<void>;
  markRead: (item: { id; source }) => Promise<void>;
  markUnread: (item: { id; source }) => Promise<void>;
  markAllRead: () => Promise<void>;
  setFilter: (filter: InboxFilterView) => Promise<void>;
}

useUnreadCount(): { count: number; isLoading: boolean; error: ChimelyError | null }
useUnseenCount(): { count: number; isLoading: boolean; error: ChimelyError | null }
```

- `unread` drives item styling. `unseen` drives the bell badge and is cleared
  by `markAllSeen` (opening the bell), independent of read state.

```tsx
usePreferences(): {
  preferences: ReadonlyArray<Preference>;   // explicit rows only; absent category = enabled
  setPreferences: (p: Preference[]) => Promise<void>;
  isLoading: boolean;
  error: ChimelyError | null;
}

// Superset for fully custom inbox UIs: snapshot + every action.
useInbox(): {
  items; counts; status; hasMore; isLoading; error;
  fetchMore; refresh; markRead; markAllRead; markAllSeen;
}
```

## `ChimelyProvider`

Provide a client to the hooks. Pass either a caller-owned `client` or a
`config` the provider constructs, connects on mount, and closes on unmount.

```tsx
import { ChimelyProvider } from '@chimely/react';

<ChimelyProvider config={{ serverUrl, environment, subscriberId, subscriberHash }}>
  {children}
</ChimelyProvider>
```

`useChimelyClient()` returns the client; it throws outside a provider.

## `@chimely/client`

```ts
import { ChimelyClient } from '@chimely/client';
const client = new ChimelyClient(config);
client.connect();
```

### `ChimelyClientConfig`

| Field | Type | Default | Notes |
|---|---|---|---|
| `serverUrl` | `string` | — | **Required.** Server origin |
| `environment` | `string` | — | **Required.** Environment slug |
| `subscriberId` | `string` | — | **Required.** Subscriber id |
| `subscriberHash` | `string` | — | HMAC hash; required in production |
| `backoff` | `BackoffConfig` | see below | SSE reconnect backoff |
| `pageSize` | `number` | `20` | 1–100 |
| `fetchFn` | `typeof fetch` | `globalThis.fetch` | Custom fetch (SSR/testing) |
| `createEventSource` | `(url) => EventSourceLike` | platform `EventSource` | Polyfill/React Native hook |

`BackoffConfig` defaults: `initialDelayMs 1000`, `maxDelayMs 30000`,
`multiplier 2`, `jitter 0.5`, `maxAttempts Infinity`.

### Methods

| Method | Returns | Notes |
|---|---|---|
| `connect()` / `close()` | `void` | Open / tear down the SSE stream |
| `getSnapshot()` | `InboxSnapshot` | Current immutable state |
| `subscribe(listener)` | `() => void` | Notified on every change |
| `refresh()` | `Promise<void>` | Conditional refetch (page 1) |
| `fetchMore({ limit? })` | `Promise<void>` | Next page |
| `markRead({ id, source })` | `Promise<void>` | Optimistic |
| `markAllRead()` | `Promise<void>` | Watermark move |
| `markAllSeen()` | `Promise<void>` | Zeroes unseen; leaves read state |
| `getPreferences()` | `Promise<Preference[]>` | — |
| `setPreferences(p)` | `Promise<Preference[]>` | — |

SSE events are hints: each one triggers a conditional REST refetch (ETag,
mostly `304`). Mutations are optimistic and roll back on failure.

### Core types

```ts
type InboxItemSource = 'notification' | 'broadcast';

interface InboxItem<TPayload = WellKnownPayload> {
  id: `notif_${string}` | `bcast_${string}`;  // prefix encodes source
  source: InboxItemSource;
  category: string;          // e.g. 'payment.failed'; Chimely never interprets payload
  payload: TPayload;
  occurredAt: string;        // RFC 3339
  read: boolean;
}

interface InboxCounts { unread: number; unseen: number }

interface Preference { category: string; channel: 'in_app'; enabled: boolean }

type ConnectionStatus =
  | 'idle' | 'connecting' | 'connected' | 'reconnecting' | 'closed';
```

`WellKnownPayload` is the default rendering shape (`title`, `body`,
`action_url`, `icon_url`, all optional); extra keys pass through verbatim
(snake_case, never case-transformed). Pass a type parameter for a typed custom
payload.
